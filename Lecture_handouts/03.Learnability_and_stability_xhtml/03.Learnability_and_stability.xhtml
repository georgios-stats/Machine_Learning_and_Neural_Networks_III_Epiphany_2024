1 Learnable Probably Approximately Correct (PAC) Learning

Note 1. We formally define the broad learning problem we will work on.

Note 2. Learning algorithms A use training data sets S which may miss characteristics of the unknown data-generating process g . Essentially, approximations (aka errors) are inevitable; some characteristics of data generating process g will be missed even if we use a very representative training set S . We cannot hope that the learning algorithm will find a hypothesis whose error is smaller than the minimal possible error.

Note 3. The PAC learning problem requires no prior assumptions about the data-generating process g . It requires that the learning algorithm A will find a predictor A ( S ) whose error is not much larger than the best possible error of a predictor in some given benchmark hypothesis class. So essentially, in practice, the researcher's effort falls on the hypothesis class H .
Definition 4. (Agnostic PAC Learnability for General Loss Functions) A hypothesis class H is agnostic PAC learnable with respect to a domain Z and a loss function ℓ : H × Z → R + , if there exist a function m H : ( 0,1 ) 2 → N and a learning algorithm A with the following property:
  • for every ε ∈ ( 0,1 ) , δ ∈ ( 0,1 ) , and distribution g over Z , when running algorithm A given training set S m ={ z 1 ,..., z m } with z i ∼ IID g for m ≥ m H ( ε , δ ) then A returns A ( S ) ∈ H such as Pr S ∼ g ( R g ( A ( S m ) ) - min h' ∈ H ( R g ( h' ) ) ≤ ε ) ≥ 1- δ (1.1)
Note 5. It may be easier to work with expectations: by using Markov inequality (1.1) becomes Pr S ∼ g ( R g ( A ( S m ) ) - min h' ∈ H ( R g ( h' ) ) ≤ ε ) ≥ 1- 1 ε E S ∼ g ( R g ( A ( S m ) ) - min h' ∈ H ( R g ( h' ) ) ) (1.2) and hence we need to work with expectations and bounded above.

Remark 6. About 1.2: The accuracy parameter ε determines how far the output rule A ( S m ) of A can be from the optimal one (this corresponds to the `approximately correct'). The confidence parameter δ indicates how likely the classifier is to meet that accuracy requirement (corresponds to the “probably” part of “PAC”).

2 Analysis of the risk based on the trade-off fitting vs stability

Note 7. Let R * = min ∀ h ( R( h ) ) be an ideal/optimal (hence minimum) Risk, and A ( S ) the learning rule from a learning algorithm A trained against dataset S . The Risk of a learning algorithm A can be decomposed as R g ( A ( S ) ) - R * = R ˆ S ( A ( S ) ) - R * + R g ( A ( S ) ) - R ˆ S ( A ( S ) ) ⏟ over-fitting (2.1)

Note 8. Over-fitting can be represented by R g ( A ( S ) ) - R ˆ S ( A ( S ) ) . However, R ˆ S ( ⋅ ) is a random variable and, here, for our computational convenience, we focus on its expectation w.r.t. S ∼ g . Hence, we provide the following (arguable) definition for over-fitting on which we base our analysis.
Definition 9. For a learning algorithm A , as a measure of over-fitting we consider the expected difference between true Risk and empirical Risk E S ∼ g ( R g ( A ( S ) ) - R ˆ S ( A ( S ) ) ) (2.2)

Definition 10. We say that learning algorithm A suffers from over-fitting when (2.2) is `too' large.

Note 11. The expected Risk of a learning algorithm A ( S ) can be decomposed as E S ∼ g ( R g ( A ( S ) ) ) = E S ∼ g ( R ˆ S ( A ( S ) ) ) ⏟ ( I ) + E S ∼ g ( R g ( A ( S ) ) - R ˆ S ( A ( S ) ) ) ⏟ ( II ) (2.3) by applying expectations in (2.1) and ignoring R * . ( I ) indicates how well A ( S ) fits the training set S , and ( II ) indicates the discrepancy between the true and empirical risks of A ( S ) . In Section 3, we argue that the over-fitting term ( II ) is directly related to a certain type of stability of A ( S ) .

Note 12. The following result connects the need for upper bounding (and minimizing this bound) the Expected Risk in (2.3) and PAC learning (Definition 4).

Proposition 13. Let A be a learning algorithm that guarantees the following:
  • If m ≥ m H ( ε ) then for every distribution g , it is E S ∼ g ( R g ( A ( S m ) ) ) ≤ min h' ∈ H ( R g ( h' ) ) + ε
Then A satisfies the PAC guarantee in Definition 4:
  • for every δ ∈ ( 0,1 ) , if m ≥ m H ( ε δ ) then Pr S ∼ g ( R g ( A ( S m ) ) - min h' ∈ H ( R g ( h' ) ) ≤ ε ) ≥ 1- δ
Proof. Let ξ = R g ( A ( S m ) ) - min h' ∈ H ( R g ( h' ) ) . From Markov's inequality Pr ( ξ ≥ E ( ξ ) / δ ) ≤ 1 E ( ξ ) / δ E ( ξ ) = δ . Namely, Pr ( ξ ≤ E ( ξ ) / δ ) =1- δ . But it is given that if m ≥ m H ( ( ε δ ) ) for every distribution g it is E ( ξ ) ≤ ( ε δ ) . So by substitution Pr ( ξ ≤ ( ε δ ) / δ ) =1- δ implies Pr ( ξ ≤ ε ) =1- δ . Now substitute back ξ and we conclude the proof.

Note 14. Hence, we aim to design a learning algorithm A ( S ) that both fits the training set and is stable; i.e, keep both ( I ) and ( II ) in (2.3) small. As seen later, in certain learning problems, there may be a trade-off between empirical risk term ( I ) and ( II ) in (2.3). Consequently, we aim to upper bound (2.3) by upper bounding ( I ) and ( II ) individually and decide which term we should benefit against the other to achieve a certain total bound.

4 Implementation in regularized loss learning problems

Definition 21. A Regularized Loss Minimization (RLM) learning problem ( H , Z , ℓ ) with a regularization function J: H → R aims at a RLM learning rule that is the minimizer h * = arg min h ∈ H ( R ˆ S ( h ) +J( h ) ) (4.1) where R ˆ S ( ⋅ ) = 1 m ∑ i=1 m ℓ ( ⋅ , z i ) , and S ={ z 1 ,..., z m } ⊂ an IID training sample.

Remark 22. The motivation for considering the regularization function J in (4.1) is to: (1.) control complexity and (2.) improve stability; as we will see later.

Note 23. Here, we make our example more specific and narrow it to the Ridge RLM learning problem (could have been LASSO etc.).

Definition 24. The Ridge RLM learning problem ( H , Z , ℓ ) , with H = W ⊂ R d , uses regularization function J( w; λ ) = λ ∥ w ∥ 2 2 with λ >0 , w ∈ W and produces learning rule A ( S ) = arg min w ∈ W ( R ˆ S ( w ) + λ ∥ w ∥ 2 2 ) (4.2)

Note 25. Recall (Term 1) how the regularization function in Ridge RLM learning problem ( H , Z , ℓ ) penalizes complexity. Essentially, it implies a sequence of hypothesis H 1 ⊂ H 2 ⊂ ... with H i ={ w ∈ R d : ∥ w ∥ 2 <i } due to duality of the corresponding minimization problem.

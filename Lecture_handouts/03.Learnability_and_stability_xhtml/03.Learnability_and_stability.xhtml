1 Learnable Probably Approximately Correct (PAC) Learning

Definition 1. (Agnostic PAC Learnability for General Loss Functions) A hypothesis class H is agnostic PAC learnable with respect to a set S and a loss function ℓ : H × Z → R + , if there exist a function m H :( 0,1 ) → N and a learning algorithm A ( ⋅ ) with the following property: for every ε ∈ ( 0,1 ) , δ ∈ ( 0,1 ) , and distribution g over Z , when running algorithm A ( ⋅ ) given m ≥ m H ( ε , δ ) i.i.d examples generated by g then A ( ⋅ ) returns h ∈ H such as Pr ( R ˆ S ( h ) - min h * ∈ H ( R ˆ S ( h * ) ) ≤ ε ) ≥ 1- δ (1.1)
Note 2. It may be easier to work with expectations: by using Markov inequality (1.1) becomes Pr ( R ˆ Z ( h ) - min h * ∈ H ( R ˆ Z ( h * ) ) ≤ ε ) ≥ 1- 1 ε E ( R ˆ Z ( h ) - min h' ∈ H ( R ˆ Z ( h' ) ) ) ⏟ δ (1.2) and hence we need to work with expectations and bounded above.

2 Over-fitting

Note 3. Following we discuss the association of stability and over-fitting. We give an (arguable) definition of over-fitting and based on this we show that: “a learning algorithm does not over-fit if and only if it is on-average-replace-one-stable”.

3 Analysis based on the fitting-stability trade-off

Note 4. Let R * = min ∀ h ( R( h ) ) be an ideal/optimal (hence minimum) Risk. The Risk of a learning algorithm A ( S ) can be decomposed as R g ( A ( S ) ) - R * = R ˆ S ( A ( S ) ) - R * ⏟ ( I ) + R g ( A ( S ) ) - R ˆ S ( A ( S ) ) ⏟ ( II ) (3.1)

Note 5. Over-fitting is reasonably quantified by R g ( A ( S ) ) - R ˆ S ( A ( S ) ) . However, R ˆ S ( ⋅ ) is a random variable and for our computational convenience we focus on expectation w.r.t. S . Hence, we provide the following (arguable) definition of over-fitting (on which we base our analysis).
Definition 6. For a learning algorithm A , as a measure of over-fitting we consider the expected difference between true and empirical risk E S ∼ g ( R g ( A ( S ) ) - R ˆ S ( A ( S ) ) ) (3.2) and we say that A suffers from over-fitting when (3.2) is `too' large.

Note 7. The expected Risk of a learning algorithm A ( S ) can be written as E S ∼ g ( R g ( A ( S ) ) ) = E S ∼ g ( R ˆ S ( A ( S ) ) ) ⏟ ( I ) + E S ∼ g ( R g ( A ( S ) ) - R ˆ S ( A ( S ) ) ) ⏟ ( II ) (3.3) (by applying expectations in (3.1) and ignoring R * ), where ( I ) indicates how well A ( S ) fits the training set D , and ( II ) indicates discrepancy between the true and empirical risks of A ( S ) . In Section 4, we argue that ( II ) measures the over-fitting and stability of A ( S ) .

Note 8. The ultimate goal in a learning problem is to design a learning algorithm A ( S ) that both fit the training set and be stable; i.e. minimizing both terms in (3.3).

Note 9. As seen later, there may be a trade-off between ( I ) and ( II ) ; expected empirical risk term and stability term.

Notation 10. Let S ={ z 1 ,..., z m } be a training sample, and A be a learning algorithm with output A ( S ) .

5 Implementation in regularized loss learning problems

Definition 17. Assume ( H , Z , ℓ ) . Regularized Loss Minimization (RLM) learning rule is the one that results as the output of jointly minimizing the empirical risk R ˆ Z ( h ) and a regularization function J: H → R that is h * = arg min ⏟ h ∈ H ( R ˆ S ( h ) +J( h ) ) (5.1)

Remark 18. The motivation for considering the regularization function J in (5.1) is to: (1.) control complexity and (2.) improve stability; as we will see later.

Note 19. We make our example more specific and narrow it to the Ridge RLM learning problem (could be LASSO, Elastic Net, etc.).

Definition 20. The Ridge RLM learning problem ( H , Z , ℓ ) , here H = W ⊂ R d , uses regularization function J( w; λ ) = λ ∥ w ∥ 2 2 with λ >0 , w ∈ W and produces learning rule A ( S ) = arg min w ∈ W ( R ˆ S ( w ) + λ ∥ w ∥ 2 2 ) (5.2)

Note 21. Recall (Term 1) that the regularization function in Ridge RLM learning problem penalizes complexity. Essentially, implies a sequence of hypothesis H 1 ⊂ H 2 ⊂ ... with H i ={ w ∈ R d : ∥ w ∥ 2 <i } .
